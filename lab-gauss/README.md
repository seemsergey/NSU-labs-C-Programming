# Вычисление определителя матрицы в поле вычетов по модулю

**mod** - простое число.
**K** - **Z<sub>mod</sub>** - кольцо вычетов по простому модулю - поле.
Операции задаются стандартно:<br>
**a, b ∈ K**<br>
a + b = (a + b) % mod<br>
a * b = (a * b) % mod<br>

*Здесь и далее все операции и числа считаются элементами поля.*

**M** - квадратная матрица n на n **M ∈ M<sub>n</sub>(K)**.

Посчитать определитель матрицы **M**, соответственно в поле **K**, это значит, что, используя арифметику поля, необходимо вычислить детерминант.
Рекомендуется использовать метод **Гаусса**, для приведения матрицы к треугольному виду, определитель такой матрицы есть произведение элементов главной диагонали. Поскольку **K** - поле, любой его ненулевой элемент обратим, поэтому алгоритм Гаусса имеет место быть. 

## Вход
*   Строка 1:
    > Натуральное число **n**, **mod** - размер матрицы и модуль соответственно<br>
    > **n** ∈ [1, 5000]; **mod** - произвольное простое число, помещающиеся в **uint32_t**<br>
    > гарантируется, что **mod** - простое
*   Следующие **n** строк:
    > в **i‑й** строке **n** натуральных чисел a<sub>i1</sub>, a<sub>2</sub>, …, a<sub>in</sub><br>
    > гарантируется, что **a<sub>ij</sub>** ∈ [0, mod)

## Выход
*   Если n не в [1, 5000]: `bad input`
*   Если сток не **n** или в строке не **n** чисел: `bad input`
*   Иначе: число из поля K (∈ [0, mod)) - определитель матрицы.

## Ограничения
*   Память:
    > **n * n * sizeof(Type)**, где **Type** беззнаковый целочисленный тип, в который помещается K.
*   Время:
    > Для маленьких тестов: **1с**.<br>
    > Для больших тестов: от **3с** до **10с**, в зависимости от n и mod.

    > В силу ограничений по времени рекомендуется использовать SIMD инструкции (AVX2 отсутствует на сервере, SSE4.2 - присутствует)<br>
    > Для оптимизации операции %, рекомендуется использовать методы быстрого деления (как вариант - <a href="https://github.com/ridiculousfish/libdivide">libdivide</a>).

# Пример 1
## Вход
<pre>
2 2
1 0
0 1
</pre>

## Выход
<pre>
1
</pre>

# Пример 2
## Вход
<pre>
2 3
0 1
1 0
</pre>

## Выход
<pre>
2
</pre>
> Пояснение: -1 = 2 mod 3

# Акинатор

# Вход
### файл `tree.txt`
	дерево вопросов и персонажей:
	- строка -- текст вопроса или персонажа
	- строка с `#` -- пустой узел
	- для узла: текст, затем рекурсивно левое (да) и правое (нет) поддеревья

### файл `in.txt`
	- ответы на вопросы: `да`, `нет` или `выход`
	- при добавлении персонажа:
		строка -- имя нового персонажа
		строка -- вопрос для отличия
		строка -- ответ `да` или `нет` для нового персонажа
	- игра запускается заново после каждого раунда, ввод `выход` завершает программу

# Выход
	если файл `tree.txt` пуст или не читается, то `Ошибка: пустой файл или ошибка чтения!`
	если ответ не `да` и не `нет`, то `Ошибка: ответ должен быть 'да' или 'нет'!`
	если имя персонажа пустое, то `Ошибка: имя персонажа не может быть пустым!`
	если узел NULL, то `Ошибка: узел не существует!`
	если ошибка записи в файл, то `Ошибка при записи в файл!`
	иначе 

### файл `tree.txt`
	перезаписывается при добавлении персонажа в том же формате.

### файл `out.txt`
#### вопросы
	`<текст вопроса> (да/нет): `
#### персонаж
	`Это <имя персонажа>? (да/нет): `
	при угадывании: `The character is guessed`
#### добавление персонажа
	`Add new character`
	`Кто это был? `
	`Как его можно отличить от <имя текущего персонажа>? `
	`Ответ на вопрос для <имя нового персонажа> - да или нет? `
	при успехе: `Character added successfully`

# Ограничения
	- длина текста вопроса или персонажа до 400 символов
	- длина ответа до 16 символов
	- время работы не более 6 секунд
	- память не более 4 МБ (включая код и данные)

# Представление дерева
```
         [Это животное?]
         /           \
    [Собака]       [Книга]
    /     \        /     \
  [#]     [#]    [#]     [#]
```

# Описание файлов `in.txt` и `out.txt`
- `in.txt`: Содержит входные данные пользователя, такие как ответы на вопросы (`да`, `нет`, `выход`) или информацию для добавления нового персонажа (имя, вопрос, ответ). Эти данные используются для взаимодействия с программой.
- `out.txt`: Содержит вывод программы, включая результаты угадывания персонажа, сообщения об ошибках.

# Пример 1
### Ввод (`in.txt`)
```
да
да
```
### Исходный файл (`tree.txt`)
```
Это животное?
Собака
#
#
Книга
#
#
```
### Выход (`out.txt`)
```
Это животное? (да/нет): 
Это Собака? (да/нет): The character is guessed
```
### Разбор Примера 1
#### Дерево
```
         [Это животное?]
         /           \
    [Собака]       [Книга]
    /     \        /     \
  [#]     [#]    [#]     [#]
```
#### Пояснение записи в `tree.txt`
Дерево записывается в `tree.txt` построчно: сначала корневой вопрос ("Это животное?"), затем левое поддерево ("Собака" и два `#` для пустых узлов), а затем правое поддерево ("Книга" и два `#` для пустых узлов). Это структура, которая определяет логику угадывания.
#### Что записывается в `in.txt`
В `in.txt` записываются ответы пользователя: `да` на вопрос "Это животное?" и `да` на вопрос "Это Собака?". Эти данные направляются в программу для прохождения по дереву.
#### Как получается `out.txt`
Программа проходит по дереву, выводя вопросы и получая ответы из `in.txt`. После ответа `да` на "Это Собака?" программа завершает угадывание и записывает в `out.txt` результат: `The character is guessed`.
#### После игры `tree.txt` не изменяется 
Так как персонаж был отгадан, а не добавлен.

### Разбор Примера 2
#### Дерево до добавления
```
         [Это животное?]
         /           \
    [Собака]       [Книга]
    /     \        /     \
  [#]     [#]    [#]     [#]
```
#### Пояснение записи в `tree.txt`
Дерево до добавления записано в `tree.txt` как: корневой вопрос ("Это животное?"), левое поддерево ("Собака" и два `#`), правое поддерево ("Книга" и два `#`). Это изначальная структура для угадывания.
#### Что записывается в `in.txt`
В `in.txt` записаны: `да` на "Это животное?", `нет` на "Это Собака?", затем данные для добавления нового персонажа — имя "Кот", вопрос "Это кот?", и ответ "да".
#### Дерево после добавления
```
         [Это животное?]
         /           \
    [Это кот?]    [Книга]
    /     \       /     \
  [Кот] [Cобака] [#]    [#]
  /  \   /   \
[#] [#] [#]  [#]
```
#### Пояснение получения `out.txt`
После ответа `нет` на "Это Собака?", программа предлагает добавить персонажа. Используя данные из `in.txt` ("Кот", "Это кот?", "да"), дерево обновляется: добавляется новый вопрос "Это кот?" как левое поддерево, с "Кот" как листом. Результат операции ("Character added successfully") и все вопросы до этого записываются в `out.txt`.
